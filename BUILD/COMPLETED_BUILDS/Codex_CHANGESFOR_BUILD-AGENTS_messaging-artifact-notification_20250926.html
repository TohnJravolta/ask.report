Proposal: Add Artifact-Notification Rule to BUILD/AGENTS.md

1. Analysis

To ensure consistent coordination, add a rule requiring agents to notify the most relevant agent(s) via the inbox/outbox messaging system whenever an artifact is created (proposal, spec, documentation change plan). This operationalizes communication and improves traceability.

2. Target

BUILD/AGENTS.md

3. Change

Insert a new bullet in the Messaging System (All Agents) section.

new_string:
- Use the file-based messaging system under `agent-coms/<agent>/{inbox,outbox}` for concise, actionable communication between agents.
- Typical uses: task handoffs, approvals/clarifications, cross-system coordination (project-plan, decisions, issues, BUILD), artifact promotions, validation summaries.
- On artifact creation (proposals, specs, doc updates), notify the most relevant agent(s) via `agent-coms/<agent>/{inbox,outbox}` with a brief actionable message linking concrete paths.
- Messages should be short, action-oriented, and include concrete file paths. Filename format: `YYYYMMDD_<short-topic>.md`.
- See protocols and template:
  - `agent-coms/README.md`
  - `agent-coms/message-template.md`

4. Rationale

  • Makes coordination explicit at artifact creation time.
  • Improves visibility and reduces stalled work due to missing notifications.
  • Keeps communication traceable and standardized via inbox/outbox flows.
