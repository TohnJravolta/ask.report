Proposal: Agent Inbox/Outbox Communication System

1. Analysis

The current communication system described in agent-coms/ is a passive, documentation-driven workflow. It defines roles and processes but lacks a mechanism for direct, asynchronous communication between agents. A directed messaging system would improve coordination efficiency.

2. Proposal

I propose the creation of an active, file-based inbox/outbox system within the agent-coms/ directory. This will allow agents to send specific, actionable messages to one another.

2.1. Proposed Directory Structure

The following directories should be created inside agent-coms/:


agent-coms/
├── gemini/
│   ├── inbox/
│   └── outbox/
├── claude/
│   ├── inbox/
│   └── outbox/
└── codex/
    ├── inbox/
    └── outbox/

2.2. Proposed Workflow

  1. To Send a Message: An agent creates a markdown file (e.g., 20250926_review-request.md) in the target agent's inbox/ folder. The sender should also place a copy in their own outbox/ for their records.
  2. To Read Messages: Each agent is responsible for monitoring their own inbox/.
  3. Message Handling: After a message has been read and acted upon, the receiving agent is responsible for deleting it from their inbox. The sending agent is responsible for deleting the corresponding message from their outbox.

3. Implementation Plan

The implementation agent (Codex) should execute the following commands to create the required directory structure.


mkdir agent-coms\gemini
mkdir agent-coms\gemini\inbox
mkdir agent-coms\gemini\outbox
mkdir agent-coms\claude
mkdir agent-coms\claude\inbox
mkdir agent-coms\claude\outbox
mkdir agent-coms\codex
mkdir agent-coms\codex\inbox
mkdir agent-coms\codex\outbox
